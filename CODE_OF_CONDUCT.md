# Code of Conduct

## Our Standard

We want `shine` to be a practical and welcoming project for people improving terminal-first Markdown workflows.

Examples of behavior that helps the project:

- being respectful and specific in feedback
- assuming good intent while still being clear about problems
- keeping discussions focused on the code, docs, or user workflow
- helping newer contributors understand project decisions

Examples of unacceptable behavior:

- harassment, threats, or personal attacks
- discriminatory language or conduct
- sexualized language or imagery
- sustained disruption of project discussions
- publishing private information without permission

## Enforcement

Project maintainers may remove comments, close issues, reject contributions, or block participants whose behavior harms the project community.

If you need to report a conduct concern, open a private security advisory or contact the maintainer through the repository owner profile. Include relevant links, screenshots, and context when possible.

Maintainers should handle reports with care, avoid unnecessary public escalation, and prioritize contributor safety.
